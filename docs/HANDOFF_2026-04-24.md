## 已修 bugs（都已生效）

### 1. Timeline 拖拽无法换 track

**根因**：`packages/remotion-ui/src/components/Timeline.tsx` 一个"优化"的 ref 缓存
（`viewportElRef`）在 `setLabelsPortalEl` 触发的重渲染里抓到了旧的 `.tracks-viewport`
DOM 节点；旧节点 detached 之后 `getBoundingClientRect()` 全 0，坐标换算变成
"item 在屏幕上的绝对 y"，`bandIdx = floor(y / 72)` 直接飙到 7、8（只有 2 条
track），路由全错。

**修法**：删 `viewportElRef` 缓存，drag 回调里每次现 `querySelector('.tracks-viewport')`。

**相关重构**：顺手把 `itemDragLogic.ts` 的 `buildPreview` 从 6 分支 zone 决策树
（A0/A/B/C/D/E + 源 track item 数特判）砍成单行 center-based：
`bandIdx = Math.floor(itemCenterY / trackHeight)` → 超出范围就建新 track，
否则目标就是 `tracks[bandIdx]`。这样和素材栏的 drop 心智一致（"鼠标在哪个 track 上就是哪个"），
再也不用补坑。

### 2. 多 item 时播放黑屏

**根因**：`packages/remotion-components/src/VideoComposition.tsx` 的 `ItemComponent`
把 `frame`（Sequence 相对帧，`useCurrentFrame()` 返回）和 `visibleFrom =
item.from`（合成绝对帧）做比较：

```ts
const isBeforeVisible = frame < visibleFrom;  // 相对帧 vs 绝对帧
```

对 `item.from = 0` 恰好两套值都是 0 蒙混过关；所有 `item.from > 0` 的 item
`hidden=true` 整段有效，外层黑底盖掉视频。所以"最左边（from=0）item OK，其他全黑"。

**修法**：`TrackComponent` 传 `visibleFrom` / `endFrame` 前转成 Sequence 相对帧；
`globalEndFrame` 参数改成预先算好的 `isGlobalEndItem` 布尔。

### 3. 前端 stuck-loading（async resolver 不重试）

**根因**：`useAsset` / `useSignedUrl` 失败后 `.catch(() => setState(undefined))`，
没有重试。任何一次 503（gateway 重启窗口、workerd 冷启动）都会永久卡住。

**修法**：加 `apps/web/lib/hooks/retryFetch.ts`，两处 hook 都换成 `fetchWithRetry`。
轻量策略：`maxAttempts=2`（1 次重试）、`capMs=1500`，只对 5xx/408/429/网络错重试，
4xx 直接透传。

## 架构决策：迁到 Vite + React Router v7

### Why

1. **开发循环**：Turbopack 问题 + OpenNext 适配层各种奇葩 bug，日常被拖累
2. **架构契合度**：你 99% 是 client-side（canvas + 视频编辑 + Loro WASM），Next.js
   的 RSC / SSR 流基本没用上，OpenNext 只是把 Next 塞进不原生的 runtime
3. **CF 官方路径**：`@cloudflare/vite-plugin` 1.0 在 **2025-04-08 GA**，React
   Router v7 是一等公民。今天去用，踩到的是后人踩过的坑（1 年稳定窗口）
4. **主站内页想要 SSR**：用户明确表示要内页 SSR（登录门禁、SEO、首屏有内容），
   所以走 SPA + 静态 `index.html` 不够，需要 loader 能访问 `env.DB / R2 /
   service bindings`

### 目标架构

```
┌─────────────────────────────────────────────────────┐
│ Worker A: web (React Router v7 SSR on CF)            │
│  ├─ /, /projects, /projects/:id → SSR + hydrate      │
│  ├─ loader 里直接 context.cloudflare.env.DB 访问      │
│  ├─ [assets] binding 托管 client hydration bundle    │
│  └─ service binding → Worker B 调后端 API            │
├─────────────────────────────────────────────────────┤
│ Worker B: api-cf (现有 Hono, 保留)                    │
│  ├─ /api/v1/*  REST                                  │
│  ├─ /sync/*    Durable Object                        │
│  ├─ /agents/*  AI                                    │
│  └─ /assets/*  R2 signed URL                         │
└─────────────────────────────────────────────────────┘

apps/gateway 撤掉 —— 职能并入 web worker 的根 route 和 api-cf 的对应路由。
```

### 参考实现

`~/Proj/playheads/` 就是**活生生的 CF-native React 栈范本**，但是它是纯 SPA
（不是 SSR）。可以抄的东西：

- `apps/web/worker/index.ts` 才 6 行（`env.ASSETS.fetch(request)`）—— SPA 兜底
  由 wrangler 本身的 `not_found_handling = "single-page-application"` 配置做掉
- Gateway 是朴素 `export default { fetch }` + service bindings，不用 Hono，
  if-else 链读性好 bundle 小
- 每个 domain 独立 worker（`apps/web` / `apps/agent` / `apps/admin` /
  `apps/landing`），service binding 互相串
- `packages/auth` 抽出 better-auth + drizzle 共享（我们当前 better-auth 还耦合
  在 Next app 里）

**我们这次不抄纯 SPA 的部分**——playheads 不需要内页 SSR，你需要。所以用 RR v7
模板而不是 plain Vite React。

### Stack 具体选择

| 层 | 用啥 |
|---|---|
| build/dev | **Vite 6** + **`@cloudflare/vite-plugin`** |
| framework | **React Router v7 SSR 模板**（`npm create cloudflare@latest -- --framework=react-router`） |
| 客户端路由 | 本身就是 React Router v7 |
| 代码分割 | `React.lazy(() => import(...))` + `<Suspense>` 替 `next/dynamic` |
| client-only 模块 | `<ClientOnly>` 包 ReactFlow / Remotion Player / dnd-kit / loro-crdt（这些依赖 `window`，不能 SSR） |
| auth | `better-auth/hono` 挂在 api-cf 或 web worker 的 `/api/auth/*`，客户端继续用 `better-auth/react` |
| 资源 | 保留现有 R2 + signed URL，`<img>` / `<video>` 直接用 |
| 部署 | 原生 `wrangler deploy`，撤 `@opennextjs/cloudflare` |

  里已经没这行了（有人删过，也加了 `backingAssetId` 映射）。没处理。

  同时播）+ ReactFlow 频繁 layout，不是服务端问题。迁移后如果改成 SSR 首屏，
  重页面依然需要 `<ClientOnly>` 兜底，不会自动变快。
